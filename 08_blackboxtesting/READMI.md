# Black Box Testing – CV Rofile Chetose

**Mata Kuliah:** Software Quality 
**Metode:** Black Box Testing

## Definisi (Sesuai Materi)
Black Box Testing adalah metode pengujian perangkat lunak yang melibatkan pengujian sistem tanpa mengetahui desain internalnya. Teknik ini berfokus pada informasi dari perangkat lunak dengan mempartisi masukan dan keluaran dari sebuah program.

## Model Black Box Testing yang Digunakan
| No | Model | Keterangan |
|----|-------|-------------|
| 1 | Boundary Value Analysis (BVA) | Menguji nilai batas (Min, Max, ±1) |
| 2 | Equivalence Partitioning (EP) | Membagi input ke kelas valid & invalid |
| 3 | Decision Table Testing | Menguji kombinasi kondisi |
| 4 | Comparison Testing | Membandingkan hasil antar versi |
| 5 | Sample Testing | Mengambil nilai dari kelas yang sama |
| 6 | Robustness Testing | Menguji input di luar spesifikasi |
| 7 | Behavior Testing | Menguji perilaku dari sudut pandang pengguna |
| 8 | Performance Testing | Mengukur kecepatan, skalabilitas, stabilitas |
| 9 | Endurance Testing | Menguji dengan pengulangan |
| 10 | Cause-Effect Testing | Menganalisis cause dan effect |
